# Workflow — Skill Loadout

## Recommended Tools

| Tool | Priority | Why |
|------|----------|-----|
| PAUL | required | Structured phases for building multi-component workflows (commands + hooks + domains) |
| Skillsmith | recommended | If the workflow produces a skill — ensures Skillsmith compliance from the start |
| CARL | recommended | If the workflow needs dynamic rules — design the domain during ideation, not after |
| BASE | optional | If the workflow touches workspace state — understand the integration surface |
